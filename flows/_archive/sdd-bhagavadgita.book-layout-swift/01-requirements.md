# Requirements: bhagavadgita.book-layout-swift (порт UI из legacy Swift)

> Version: 1.0  
> Status: DRAFT  
> Last Updated: 2026-04-29

## Problem Statement

Текущее Flutter-приложение `app/bhagavadgita.book` не содержит (или содержит частично) UI-вёрстку и визуальные ассеты, которые уже реализованы в legacy iOS-приложении на Swift (`legacy/legacy_bhagavadgita.book_swift/Gita`). Нужно перенести в Flutter:

- экраны и компоновки (включая состояния: пусто/загрузка/ошибка),
- визуальные элементы (иконки/картинки/сплэш),
- базовые UI-стили (цвета/типографика/отступы),

и сделать адаптивную вёрстку под разные размеры экранов (phone/tablet/desktop/web), сохранив визуальный смысл legacy UI.

## User Stories

### Primary

**As a** читатель Бхагавад-гиты  
**I want** иметь удобный, читаемый и знакомый интерфейс приложения на всех устройствах  
**So that** я могу быстро находить главы/шлоки/закладки и комфортно читать/делиться контентом

### Secondary

- **As a** пользователь планшета/десктопа  
  **I want** адаптивную компоновку (широкие экраны, split-view)  
  **So that** контент не выглядит растянутым и используется пространство экрана

## Acceptance Criteria

### Must Have

1. **Given** приложение запускается на iPhone/Android phone  
   **When** я открываю основные экраны (список глав/шлоки/закладки/настройки/поиск/цитаты)  
   **Then** UI корректно отображается без переполнений/клиппинга и соответствует смыслу legacy-вёрстки.

2. **Given** приложение запускается на iPad/планшете/десктопе  
   **When** я открываю те же экраны  
   **Then** компоновка адаптируется (ограничение ширины контента, колонки/side panel где уместно), без потери функциональности.

3. **Given** в legacy Swift используются изображения/иконки  
   **When** я использую Flutter-эквивалентные экраны  
   **Then** ассеты перенесены и подключены через `pubspec.yaml` и отображаются корректно (включая resolution-aware варианты, где применимо).

4. **Given** пользователь меняет системный размер шрифта/настройки доступности  
   **When** я читаю шлоки/переводы/комментарии  
   **Then** текст остаётся читаемым, не ломает верстку (в разумных пределах), скролл/перенос строк работает.

### Should Have

- Аккуратная общая дизайн-система (токены: spacing, радиусы, типографика) для консистентности.
- Тёмная тема, если в проекте уже заложена архитектура темизации (иначе — отложить).

### Won't Have (This Iteration)

- Перенос всей бизнес-логики legacy (если она не нужна для UI-слоя).
- Полная 1:1 пиксель-перфект копия, если это противоречит Flutter-гайдлайнам/адаптиву (приоритет — качество UX и адаптив).

## Constraints

- **Technical**: перенос выполняется внутри `app/bhagavadgita.book` (Flutter/Dart), без встраивания iOS storyboard/xib.
- **Assets**: ассеты должны жить в `app/bhagavadgita.book/assets/` и быть перечислены в `pubspec.yaml`.
- **Platform**: минимум iOS + Android; желательно web/desktop без UI-regressions.
