# Requirements: vdd-bhagavadgita.book-audioplayer

> Version: 0.1  
> Status: DRAFT  
> Last Updated: 2026-04-30

## Problem Statement

Нужен «идеальный» аудиоплеер для Bhagavadgita.book: быстрый, надёжный, офлайн‑дружелюбный, эстетичный и одинаково удобный на **телефоне, планшете и десктопе**, учитывающий специфику контента (шлоки, главы, «санскрит / перевод», автопереходы) и привычки пользователей (слушать в фоне, в дороге, во время чтения).

Источники аудио в текущем проекте:

- **Предустановленные статические файлы (offline‑bootstrap)**:
  - `app/bhagavadgita.book/assets/audio/ru/chapter_1_ru.mp3`
  - `app/bhagavadgita.book/assets/audio/sanskrit/chapter_1_sanskrit.mp3`
- **Онлайн‑скачивание всех глав** через параметры в `.env`:
  - `AUDIOVEDA_RUSSIAN_URL`
  - `AUDIOVEDA_SANSKRIT_URL`
  - `AUDIOVEDA_USERNAME`
  - `AUDIOVEDA_PASSWORD`

Референсы в текущем репозитории показывают прошлые решения:

- iOS (Swift): `AVPlayer` + простой прогресс/детект завершения по таймеру, встраиваемый `ShlokaPlayerView`, настройки «аудио санскрит/перевод + autoplay».
- Android (Java): `MediaPlayer` + audio focus, скачивание аудио через системный `DownloadManager` и таблицу `AudioState` (статусы «скачано/в очереди/ошибка»).

Часть названных референсов (avadhuta, cookbook/gitanjali/sgg‑ru swift) в этом репозитории не найдена — требования ниже учитывают их как будущие источники идей, но не опираются на конкретные их детали.

## Users / Personas

- Читатель‑слушатель: читает шлоку и хочет «в одно касание» слушать санскрит/перевод, переключаться и продолжать.
- Слушатель‑в‑фоне: ставит плейлист (шлоки/глава), управляет из lockscreen/гарнитуры/уведомлений.
- Офлайн‑пользователь: заранее скачивает аудио (по языкам/книгам/главам), контролирует объём и качество.
- «Desktop‑worker»: слушает за компьютером, хочет мини‑плеер/горячие клавиши/управление из системного tray/menubar.

## 108 требований/идей

1. Плеер должен быть **единым модулем**, используемым из экрана чтения, списка глав и отдельного «Now playing».
2. Поддержать 2 дорожки для одного контента: **Sanskrit** и **Translation** (как в legacy настройках).
3. Поддержать «трек» как шлоку и «сессию» как (глава → шлоки) или (выбранные шлоки → очередь).
4. Дать режим **Autoplay**: после завершения текущей шлоки запускать следующую (аналог legacy `autoPlayAudio`).
5. Дать режим «повтор»: repeat‑one (текущая шлока) и repeat‑all (очередь).
6. Дать режим «перемешать» для очереди (shuffle) с сохранением seed для воспроизводимости.
7. Сделать «плейлист главы» как первоклассный объект (создание/воспроизведение/сохранение).
8. Дать «продолжить слушать» с сохранением позиции по каждому треку.
9. Сохранять прогресс в фоне и после перезапуска приложения.
10. При переходе между шлоками сбрасывать/переносить позицию согласно настройке (по умолчанию: с 0).
11. Реализовать **мгновенную реакцию UI** (optimistic): кнопка Play меняется сразу, даже если буферизация ещё идёт.
12. Отображать состояния: буферизация, воспроизведение, пауза, ошибка, нет сети, нет файла.
13. Уметь «продолжить» после временной потери audio focus / interruption (звонок/уведомление/другой звук).
14. Поддержать управление из гарнитуры/кнопок (play/pause/next/prev).
15. Поддержать lockscreen/notification controls (OS‑native).
16. Показать метаданные трека: глава, номер шлоки, язык дорожки, название, длительность.
17. Для цитат/шлок — генерировать краткий subtitle (например «Chapter 2, Shloka 47»).
18. Поддержать **скорость** (0.75×, 1×, 1.25×, 1.5×, 2×) с сохранением выбора.
19. Поддержать «skip вперед/назад» фиксированными шагами (например 10/30 секунд, настраиваемо).
20. Для длинных дорожек (если появятся) добавить «глава как один трек» (опционально).
21. Поддержать **seek** по прогресс‑бару (с безопасной деградацией для стрима).
22. Визуально показывать прогресс как тонкую линию (как в legacy `ShlokaPlayerView`) и/или slider в полноэкранном режиме.
23. Встроенный плеер на экране шлоки должен занимать минимум места и не ломать типографику.
24. Плеер на экране шлоки должен иметь 2 состояния: «свернутый» (только play + прогресс) и «расширенный» (скорость, seek, переключатели).
25. На телефоне в портрете: bottom mini‑player + свайп вверх в full player.
26. На телефоне в ландшафте: компактный режим, не перекрывающий текст (например поверх с прозрачным фоном/или в колонке).
27. На планшете: двухпанельный layout — текст слева, плеер/очередь справа (как «reading + listening»).
28. На планшете поддержать drag&drop добавления шлок в очередь (если платформа позволит).
29. На десктопе: выделенный Now Playing экран + всегда видимый мини‑плеер в заголовке/нижней панели.
30. На десктопе добавить глобальные горячие клавиши (play/pause/next/prev).
31. На десктопе рассмотреть режим **tray/menubar**: иконка, мини‑контролы, «показать/скрыть», быстрый выбор дорожки.
32. Tray‑режим должен работать без постоянного открытого окна (если возможно для Flutter desktop).
33. В tray‑режиме показать текущую шлоку/главу и кнопку play/pause минимум.
34. В tray‑режиме предусмотреть «Quit» и «Open app».
35. Поддержать системные media keys (если доступно на платформе).
36. Визуальная система плеера должна следовать теме приложения (цвет `red1`, нейтральные `gray*` как в legacy).
37. Дизайн должен учитывать крупный текст/Devanagari (не ломать высоты/переполнения).
38. Добавить «вибро/звук клика» (haptic) на основные кнопки (мобилки), с настройкой отключения.
39. Показать «что именно сейчас звучит»: бейдж дорожки (Sanskrit/Translation) рядом с заголовком.
40. Позволить мгновенно переключать дорожку, сохраняя позицию если длительности совпадают/близки (или сбрасывать по правилу).
41. Если дорожка отсутствует для шлоки — показывать disabled‑состояние и подсказку «нет аудио».
42. Если пользователь включил дорожку в настройках, но аудио не скачано — предложить скачать (как legacy alert).
43. Скачивание аудио должно быть отдельным слоем (download manager) и не блокировать UI.
44. Поддержать «скачать всё аудио для выбранной дорожки» (перевод/санскрит) для книги/главы.
45. Поддержать «скачать только текущую главу» (экономия места).
46. Поддержать «скачать только избранное/закладки».
47. Показать оценку размера загрузки перед подтверждением.
48. Показать прогресс загрузки (общий + по файлам) в настройках и в плеере.
49. Поддержать pause/resume загрузок.
50. Поддержать автоматическое возобновление после перезапуска приложения.
51. Поддержать отмену загрузки и очистку частично скачанного.
52. Поддержать «удалить аудио» для дорожки/книги (как legacy delete prompt).
53. Уметь чистить кэш/временные файлы и показывать сколько места освобождено.
54. Настройки «аудио санскрит/перевод» должны быть доступны и как глобальные, и как быстрые в плеере.
55. Настройка Autoplay должна быть доступна в плеере (переключатель) и в общих настройках.
56. Добавить настройку «играть в фоне» (по умолчанию включено) с объяснением.
57. Добавить настройку «останавливать при открытии текста» (если пользователь хочет только читать).
58. Добавить настройку «прерывать при навигации» (или продолжать).
59. Добавить «таймер сна» (sleep timer): 5/10/15/30/60 минут, «до конца шлоки».
60. Добавить «повторить последнюю фразу» (быстрый jump назад на 3–5 сек).
61. Добавить режим «петля по диапазону» (A‑B loop) для практики произношения.
62. Для A‑B loop сделать сохранение пресетов на конкретных шлоках.
63. Поддержать «заметки по аудио» (timestamped notes) для обучения.
64. Сделать кнопку «поделиться» текущей шлокой с отметкой «слушаю» (без нарушения авторских прав аудио).
65. Не позволять шарить прямой файл аудио, если лицензии не позволяют (policy requirement).
66. Добавить «умный старт»: если пользователь на экране шлоки нажимает play, запускать выбранную дорожку и показывать мини‑плеер.
67. Если пользователь скроллит, мини‑плеер не должен перекрывать системные элементы и bottom nav.
68. Плеер должен корректно работать при смене ориентации без сброса.
69. Плеер должен корректно работать при смене темы (light/dark) без мерцаний.
70. Поддержать доступность: крупные тапы, правильные label’ы для screen reader, фокус‑навигация на десктопе.
71. Для VoiceOver/TalkBack проговаривать: «Play/Pause, Chapter X, Shloka Y, Sanskrit/Translation, progress 20%».
72. Поддержать управление с клавиатуры (Tab/Enter/Space) на десктопе.
73. Встроить логику **audio focus / interruptions** по‑взрослому (не только таймером как legacy iOS).
74. На iOS: использовать корректные категории/опции аудиосессии (background, mixing/ducking по требованиям).
75. На Android: использовать современные API (предпочтительно ExoPlayer/Media3) вместо сырого `MediaPlayer` (как improvement).
76. На Android: интегрироваться с MediaSession для корректных контролов.
77. На всех платформах: уметь восстанавливаться после ошибки декодирования/файла (fallback + сообщение).
78. Если файл повреждён — предложить «перескачать».
79. Поддержать несколько качеств (если есть): low/normal/high, с авто‑выбором по сети/месту.
80. Поддержать стриминг/онлайн‑источник (если сервер предоставляет) как опцию, но офлайн должен быть first‑class.
81. Для стрима обеспечить безопасный seek и прогресс (учитывая legacy баг‑фиксы seek/stream).
82. Ввести единый формат идентификаторов треков (bookId/chapter/shloka/language/voice/version).
83. Поддержать версии аудио (например новые озвучки) без поломки старых скачиваний.
84. Поддержать несколько голосов/дикторов (если появятся) и их переключение.
85. Добавить «тональность/эквалайзер» только если это не перегрузит UX (скорее v2).
86. Вести статистику (локально): сколько слушают, где бросают, какие дорожки предпочитают.
87. События аналитики должны быть приватными и отключаемыми (opt‑out).
88. События: play/pause/complete, переключение дорожки, скачивание/удаление, speed change, sleep timer.
89. Плеер не должен тратить батарею в паузе (таймеры/стримы должны останавливаться).
90. Таймер прогресса должен быть event‑based где возможно (не polling 0.25s как legacy) или адаптивным.
91. UI прогресса должен обновляться плавно, но экономно (throttle).
92. Кнопки должны иметь состояния disabled/loading и не допускать гонок (double tap).
93. Поддержать очередь «Next up»: показать следующие 3–5 шлок.
94. Разрешить пользователю переставлять элементы очереди.
95. Добавить «играть с этой шлоки до конца главы».
96. Добавить «прыжок к текущей шлоке» из любого места (быстрый shortcut).
97. В full player добавить вкладки: «Трек», «Очередь», «Скачать», «Настройки».
98. В режиме чтения показывать «auto‑scroll / highlight» текущей строки (если будет тайминг; иначе — v2).
99. При окончании трека — мягкая анимация перехода и опциональный «дзынь» (настраиваемо).
100. Поддержать «предзагрузка следующего трека» (preload) для бесшовного автоплея.
101. При плохой сети — стратегия: снижать качество/перейти на офлайн‑файл/показать CTA «скачать».
102. При недостатке места — предупредить и предложить очистку/выбор меньшего набора.
103. В настройках показать управление хранилищем: общий размер, по дорожкам, по главам.
104. Хранить файлы в предсказуемой структуре и уметь мигрировать схему хранения между версиями.
105. Добавить «диагностику» для поддержки: экспорт логов плеера/скачиваний (без персональных данных).
106. Добавить «режим разработчика» (скрытый): показывать ID трека, источник (offline/stream), буфер.
107. Обеспечить совместимость UI/UX между платформами: одинаковые иконки, термины, поведения.
108. Определить «идеальный минимум» (MVP): play/pause, прогресс, автоплей, выбор дорожки, офлайн (включая предустановленный Chapter 1) + скачивание всех глав онлайн, lockscreen/notification controls, tablet+desktop layouts, (desktop tray — если технически возможно).

## Constraints

- **Platform**: iOS/Android обязаны поддерживать фон и системные контролы; desktop (macOS/Windows/Linux) — минимум базовый плеер, расширения (tray) по возможностям Flutter desktop.
- **Offline-first**: слушать без сети после скачивания.
- **Content coupling**: треки привязаны к book/chapter/shloka + языку дорожки.
- **Bootstrap offline**: минимум один пакет аудио должен работать из коробки без сети (предустановленные mp3 для Chapter 1: ru/sanskrit).
- **Credentials**: учётные данные для AudioVeda берутся только из `.env`, не логируются, не попадают в аналитику и не коммитятся.
