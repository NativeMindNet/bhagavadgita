# Команда: /translate.sanscrit

## Назначение

Автоматический перевод глав Бхагавад-гиты (шлоки + словарь) на азиатские языки с последующим сохранением результата.

## Синтаксис

```
/translate.sanscrit [путь_к_файлу]
```

### Параметры

| Параметр | Обязательный | Описание |
|----------|--------------|----------|
| `путь_к_файлу` | Да | Путь к JSON файлу для сохранения результата |

## Примеры использования

```bash
# Перевод главы 1
/translate.sanscrit data/translations/chapter-01-translations.json

# Перевод главы 2
/translate.sanscrit data/translations/chapter-02-translations.json
```

## Процесс выполнения

### 1. Извлечение данных

Агент автоматически извлекает:
- **Шлоки**: из `data/chapters/chapter-XX.json`
- **Словарь**: из `data/vocabulary/vocab_XX.json`

### 2. Перевод

Агент переводит на **5 языков**:
- **Korean (ko)** — 한국어
- **Thai (th)** — ภาษาไทย
- **Chinese Simplified (zh-CN)** — 简体中文
- **Chinese Traditional (zh-TW)** — 繁體中文
- **Japanese (ja)** — 日本語

### 3. Источники для перевода

**Приоритет источников**:
1. **Русский (ru)** — основной источник смысла
2. **Английский (en)** — второй источник смысла
3. **Немецкий (de), Испанский (es)** — дополнительный контекст
4. **Санскрит** — ТОЛЬКО для уточнения терминов и имён

### 4. Транслитерация

Агент добавляет транслитерацию санскритских слов на языках назначения:
- **Тайская** — фонетическая транскрипция
- **Китайская** — пиньинь
- **Корейская** — хангыль
- **Японская** — ромадзи + кана

### 5. Сохранение результата

Результат сохраняется **немедленно** после обработки:

```json
{
  "chapter": 1,
  "chapterTitle": {
    "ko": "...",
    "th": "...",
    "zh-CN": "...",
    "zh-TW": "...",
    "ja": "..."
  },
  "slokas": {
    "1.1": {
      "ko": "...",
      "th": "...",
      "zh-CN": "...",
      "zh-TW": "...",
      "ja": "..."
    }
  },
  "vocabulary": {
    "1": {
      "meaning": {
        "ko": "...",
        "th": "...",
        "zh-CN": "...",
        "zh-TW": "...",
        "ja": "..."
      },
      "transliteration": {
        "th": "...",
        "zh-CN": "...",
        "ko": "...",
        "ja": "..."
      }
    }
  }
}
```

## Правила выполнения

### Sequential Execution (Последовательное выполнение)

- **Блокировка**: Следующая глава **не переводится**, пока не сохранён результат текущей
- **Immediate Save**: Результат сохраняется **сразу** после обработки агентом
- **No Manual Intervention**: Полностью автоматизированный процесс

```
Chapter 1 → Agent → Save → Validate → Chapter 2 → Agent → Save → ...
```

### Error Handling

| Ошибка | Действие |
|--------|----------|
| Файл не найден | Логирование, переход к следующей главе |
| JSON валидация не прошла | Логирование, повторная попытка |
| Агент не ответил | Логирование, повторная попытка (3 раза) |

## Выходные данные

### Формат результата

JSON файл со следующей структурой:

```json
{
  "chapter": <number>,
  "chapterTitle": { <language>: <text> },
  "slokas": { <sloka_number>: { <language>: <text> } },
  "vocabulary": { <id>: { 
    "meaning": { <language>: <text> },
    "transliteration": { <language>: <text> }
  } }
}
```

### Валидация

После сохранения агент автоматически проверяет:
- ✓ JSON валиден
- ✓ Все 5 языков присутствуют
- ✓ Все шлоки переведены
- ✓ Весь словарь переведён
- ✓ Транслитерация добавлена

## Интеграция с SDD Flow

Эта команда является частью flow:
- `flows/sdd-gitabook-translation-vocabulary-json/`

Используется в Phase 3: **Automated Translation**

## Связанные команды

| Команда | Описание |
|---------|----------|
| `/sdd resume sdd-gitabook-translation-vocabulary-json` | Возобновить flow перевода |
| `/sdd status` | Показать статус всех active flows |

## Ограничения

- **Encoding**: UTF-8 для всех файлов
- **Size**: Один файл перевода < 10MB
- **Timeout**: 30 минут на главу
- **Languages**: Только ko, th, zh-CN, zh-TW, ja
